HƯỚNG DẪN SỬ DỤNG NẰM Ở DƯỚI

Xin chào, bạn có thể gọi tôi là Egg, tôi là một AI Builder nghiệp dư đã có một ít kinh nghiệm trước đây với dự án "CallByEgg", bạn có thể xem nó ở phần dự án của tôi trong Github. Tôi đã đóng băng dự án đó khá lâu, và giờ đây tôi trở lại với dự án "Transferbyegg".

-Truyền file tưởng đơn giản, nhưng thực tế nếu bạn có tiêu chuẩn hoặc nhu cầu cao hơn một chút thì lại khá rắc rối:
+ Bạn muốn gửi ảnh từ thiết bị này sang thiết bị khác nên bạn gửi ảnh qua các ứng dụng nhắn tin cho gọn → ảnh bị nén, vỡ nét
+ Bạn upload lên Google Drive, OneDrive, Mega, Dropbox → bản miễn phí phần lớn đều bị giới hạn dung lượng, download còn có thể bị chặn nếu vượt mức, và đối với bản thân tôi, việc dùng các nền tảng này chỉ để truyền một bức ảnh vài MB không phải là một giải pháp tối ưu về mặt thời gian
+ Bạn dùng app/web P2P → nếu trong mạng bạn có nhiều người cần nhận file thì có bao nhiêu người bạn phải upload lại bấy nhiêu lần (sẽ rất mất thời gian nếu là file nặng). Thậm chí là nếu như trong hai người, có một người mạng yếu thì tốc độ tải của cả hai sẽ đều chậm

1.Hãy chọn đúng phiên bản hệ điều hành và tải về (có thể bị trình duyệt cảnh báo là không an toàn nhưng app này hoàn toàn PUBLIC SOURCE, nên bạn hoàn toàn có thể kiểm tra)

Máy bạn sẽ trở thành host, vì vậy vị trí ổ cứng bạn chọn có bao nhiêu dung lượng trống thì đó là lượng dung lượng mà bạn và mọi người có thể truyền (tất cả các file được tải lên trong trang web sẽ lưu vào ổ cứng của bạn)
4.nếu bạn muốn mọi người hoặc thiết bị khác của bạn truy cập nhanh để truyền file thì hãy quét mã QR trong terminal (bảng điều khiển màu đen) hoặc nhập http://:3000 vào trình duyệt để truy cập (có hiển thị trong terminal) nếu không thể quét
5.Hãy sử dụng nó để truyền các file bạn muốn *ở trên máy tính bạn có thể thả file trực tiếp vào để truyền*
6.Sau khi dùng xong, ĐỪNG tắt web bằng các nhấn tắt Terminal (bảng điều khiển màu đen), mà truy cập giao diện web của http://localhost:3000/ hãy cuộn xuống dưới cuối trang và bạn sẽ thấy nút "Close Website and Delete All Data" Hãy nhấn nút đó để trang web được đóng và các file được lưu tạm ở vị trí bạn chọn sẽ được xóa sạch (nếu bạn tắt bằng cách đóng Terminal thì các file đó sẽ bị giữ nguyên, gây ra nặng ổ cứng không đáng có)

THE INSTRUCTIONS ARE AT THE BOTTOM

Hello, you can call me Egg. I'm an amateur AI Builder with some prior experience with the "CallByEgg" project, which you can view in my Github projects section. I froze that project quite a while ago, and now I'm back with the "Transferbyegg" project.

- File transfer seems simple, but in reality, if you have higher standards or needs, it can be quite complicated:
+ You want to send photos from one device to another, so you send them through messaging apps for convenience → the images get compressed, lose quality
+ You upload to Google Drive, OneDrive, Mega, Dropbox → the free versions mostly have storage limits, downloads can be blocked if exceeded, and for me personally, using these platforms just to transfer a few MB image isn't a time-optimal solution
+ You use P2P apps/websites → if there are multiple people in your network who need to receive the file, you have to upload it as many times as there are recipients (which would be very time-consuming for large files). Even between two people, if one has a weak network connection, the transfer speed for both will be slow
I think you've faced these problems before, you're not alone in these limitations, and neither am I. So I created this app, simply put it's an app that allows you to transfer files directly with devices connected to the same network (LAN). And the transferable capacity will be the hard drive space of the machine chosen as the host for everyone to use (see step 3 of the user guide for more details).
And I chose the most practical way to implement it, which is leveraging AI. I'm not a coder, I'm not good at coding, but I used AI to bring my idea to life. The entire website you're viewing and the application you're learning about were created entirely using Qwen, ChatGPT, DeepSeek, and Claude. It might sound unusual, but that's exactly my goal - I want to pursue a solution that's built quickly, free, practical, doesn't require too much effort, and serves the file transfer needs within my internal network, hoping it will also meet everyone else's needs.

This isn't a story about who's better than whom. I'm not trying to claim how my app compares to big players like Google Drive, OneDrive, or Dropbox - they have their strengths, especially when you need long-term file storage or paid features, etc. I'll admit that I can't compete with them. Here, I just want to offer a new choice: simple, quick, useful, and most importantly, free for everyone. I hope it will be useful and bring value to everyone.

USER GUIDE:

1. Please select the correct operating system version and download (your browser might warn that it's unsafe, but this app is completely PUBLIC SOURCE, so you can verify it)
2. Run the application
3. Go to http://localhost:3000/ to specify the location where temporary files will be stored when people transfer files through the web. Only after completing this step can others access it via QR code. Your machine will become the host, so the available space on the hard drive you select will determine the total capacity that you and others can use for transfers (all files uploaded through the website will be stored on your hard drive)
4. If you want others or your other devices to quickly access for file transfer, scan the QR code in the terminal (the black control panel) or enter http://:3000 (in termial) in your browser if you can't scan
5. Use it to transfer the files you want *on your computer, you can drag and drop files directly to transfer*
6. After use, DO NOT close the web by closing the Terminal (the black control panel). Instead, access the web interface at http://localhost:3000/, scroll to the bottom of the page, and you'll see a "Close Website and Delete All Data" button. Click that button to properly close the website and delete all temporarily stored files in your selected location (if you close by terminating the Terminal, those files will remain, causing unnecessary hard drive usage)
